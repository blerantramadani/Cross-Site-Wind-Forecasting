# Cross-Site-Wind-Forecasting

Physics-informed Digital Twin and cross-site evaluation framework for LSTM-based wind power forecasting.

This repository contains the complete code accompanying the paper:

> **B. Ramadani and V. Fustic**, "When Does Recurrent Capacity Help? A Cross-Site Evaluation of LSTM-Based Wind Power Forecasting on Synthetic and Operational SCADA Data," *IEEE Access*, 2026.

It reproduces every quantitative result reported in the paper: the synthetic Digital Twin experiments, the operational SCADA evaluations on the Kelmarsh and Penmanshiel wind farms, the feature- and lookback-ablation studies, the noise-floor diagnostic, and the Diebold-Mariano significance tests.

---

## 1. Overview

The framework is organized in three tiers:

1. **A physics-informed Digital Twin** that generates synthetic wind power time series from a stationary Ornstein-Uhlenbeck wind-speed process, a temperature-dependent cubic power-conversion model, and a dynamic air-density formulation derived from the ideal gas law.
2. **Four forecasting models** evaluated under identical training and evaluation protocols: persistence, linear regression, a multilayer perceptron (MLP), and a long short-term memory (LSTM) network.
3. **Statistical significance testing** using the Diebold-Mariano test with the Harvey-Leybourne-Newbold small-sample correction, applied to five-seed averaged predictions.

## 4. Datasets

The two operational datasets are publicly available under CC-BY-4.0 and are **not** redistributed here. Download them from Zenodo and place the preprocessed hourly CSV files in the `data/` directory:

- **Kelmarsh wind farm data** — C. Plumley, Zenodo, 2022. https://doi.org/10.5281/zenodo.5841834
- **Penmanshiel wind farm data** — C. Plumley, Zenodo, 2022. https://doi.org/10.5281/zenodo.5946808

The synthetic dataset is generated locally by the Digital Twin engine (Section 5, step 1) and requires no download.

Expected file layout in `data/`:

```
data/
├── Synthetic_Bogdanci_OU.csv        # generated by digital_twin_engine_v2_1.py
├── Synthetic_Bogdanci_Noise.csv     # generated by digital_twin_engine_v2_1.py
├── Kelmarsh_T1_2018_hourly.csv      # derived from Zenodo 5841834
└── Penmanshiel_T01_2018_hourly.csv  # derived from Zenodo 5946808
```

Each preprocessed CSV contains hourly records with the columns `Timestamp`, `WindSpeed_m_s`, `Temperature_C`, and `ActivePower_kW`.

## 5. Reproducing the results

**Step 1 — Generate the synthetic Digital Twin data:**

```bash
python3 digital_twin_engine_v2_1.py
```

This produces the physics-consistent (Ornstein-Uhlenbeck) and uncorrelated-noise datasets in `data/`.

**Step 2 — Run the full pipeline on each dataset:**

```bash
python3 forecasting_pipeline_v2_3.py            # synthetic (default)
python3 forecasting_pipeline_v2_3.py kelmarsh   # Kelmarsh SCADA
python3 forecasting_pipeline_v2_3.py penmanshiel # Penmanshiel SCADA
```

Each run trains all four models with five random seeds, performs the feature- and lookback-ablation studies, computes the noise floor, and writes per-dataset results to `results/`.

**Step 3 — Run the Diebold-Mariano significance tests on five-seed averaged predictions:**

```bash
python3 dm_test_5seeds.py
```

This reproduces Table VIII of the paper.

## 6. Key configuration

All experiments use a single, uniform configuration (defined at the top of `forecasting_pipeline_v2_3.py`):

| Parameter | Value |
|-----------|-------|
| Lookback window | 48 hours |
| Epochs | 50 (early stopping, patience 5) |
| Batch size | 32 |
| Optimizer | Adam (learning rate 0.001) |
| Seeds | 5 (0–4) |
| Test window | final 1,000 hours |
| LSTM | 128 → 64 units, dropout 0.05 |
| MLP | 128 → 64 units, dropout 0.05 |

nRMSE is normalized per installation by rated capacity: 2.3 MW for the synthetic Digital Twin (Bogdanci class) and 2.05 MW for the Kelmarsh and Penmanshiel turbines (Senvion MM92).
